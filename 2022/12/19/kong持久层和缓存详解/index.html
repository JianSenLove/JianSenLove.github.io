数据迁移文件

migrations

–init.lua (必须)

–000_base_ (必须)

–001_<from-version>_to_ (可选)

init为初始化文件,000，001则为版本文件，一般000为创建数据库，当数据库字段或者其他信息需要更新时，创建一个新的版本控制文件001(以此类推)记录，并在init添加版本文件名即可

迁移文件语法(postgres)

主要有两种更新方式,up和teardown ,up为sql声明,teardown为一个函数，通过调用connector的query来执行sql语句

  • up是原始 SQL/CQL 语句的可选字符串。这些语句将在执行时kong migrations up执行。
  • teardown是一个可选的 Lua 函数，它接受一个connector参数。这样的连接器可以调用query方法来执行 SQL/CQL 查询。执行是由 kong migrations finish

数据缓存

缓存有两个

1.每个nginx工作进程的Lua memory cache —— local to an Nginx worker process

2.所有工作进程共享的缓存区——Shared memory cache (SHM) - local to an Nginx node

当从数据库请求数据时，会在两个缓存中都存储数据，当同一个工作进程再次访问时，使用lua memory chche，也就是自己进程的缓存，当别的工作进程访问该字段时，访问共享缓存区
