Kong Admin Api的扩展(core)

发表于2023-01-10|更新于2023-07-14|docker和kong
|阅读量:

相关资料几乎没有(可能这种扩展需求很少的原因?)

​ 看了一个礼拜的源码，终于成功扩展了原有的核心api。

网上我能找到的资料，全是在讲admin api的使用或者插件api的扩展，笑了，真肤浅(全是cp官方文档的)

官方文档也没有讲Kong的源码结构和加载顺序，跟别说核心admin api的扩展了

版本差异问题

​ 首先强调,kong 2.0,2.3,3.0这三个版本的扩展有一些差异，尤其是2.0到2.3的转变，几乎原本的schema下的表都做了修改，因为引用的schema.typeof包做了修改，例如:2.0字段直接使用typeof.name，而2.3使用typeof.utf8_name。而且2.3左右的版本开始使用了workspace字段以及相关设置。等等。

​ 为什么要关注版本差异，好像并不要多解释，我一开始以为版本升级只是做了扩展，并没有想到对原来的文件和字段也重写了,导致我白白浪费了一个礼拜还没有完成目标！而且还是不断制作镜像，启动容器，查看日志，直到调试成功，真难。
